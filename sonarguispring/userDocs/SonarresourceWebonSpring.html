  1. Workflow

    1. Activate the sonar resource described in LabIss | Sonar as resource on a PC (address 192.168.1.pc).
    2. Working in this project (sonarguispring), set the proper values in the configuration file configurator.kt.
      Now, activate the Spring application SonarguigpringApplication.kt on the PC, open a browser on localhost:8083 (or 192.168.1.pc:8083) and send some command to the basicrobot.


    3. Activate a CoAP client working as an observer of the basicrobot resource, e.g. actorQakCoapObserver.kt, move the basicrobot and notice that it shows, after a command, the same values that appears in the Resource state section of the web-page.
    4. Optionally, change the HTML page to be displayed, by changing the String htmlPage in RobotController.java. For example, the previous page (robotGuiSocket) sends commands by using a socket.io, while the following one (robotGuiPost) does use the conventional POST HTTP-verb :
